# 환경 변수 설정 가이드

## 1. Supabase 프로젝트 확인

먼저 Supabase에서 다음 정보를 찾아야 합니다:

### Supabase 대시보드에서 찾는 방법
1. https://app.supabase.com 방문
2. 왼쪽 사이드바에서 본인의 프로젝트 선택
3. "Settings" (설정) > "API" 클릭
4. 아래 정보를 복사하여 저장:
   - **Project URL**: `https://xxxxx.supabase.co`
   - **Anon Key**: `eyJhbGci...` (긴 텍스트)

---

## 2. .env.local 파일 업데이트

프로젝트 루트 폴더(package.json이 있는 폴더)의 `.env.local` 파일을 열고,
다음 두 줄을 추가하세요:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
```

> ⚠️ 주의: `NEXT_PUBLIC_`로 시작하는 변수는 브라우저에 노출됩니다.
> 민감한 정보(비밀 키)는 절대 여기에 넣으면 안 됩니다.

---

## 3. Google OAuth 설정 (Supabase에서)

### 3-1. Supabase 대시보드에서 설정
1. "Authentication" (인증) 메뉴 클릭
2. "Providers" (제공자) 선택
3. "Google" 찾기
4. Google을 **활성화** 상태로 전환
5. 아래 정보 입력:
   - **Client ID**: Google Cloud Console에서 얻기 (아래 참고)
   - **Client Secret**: Google Cloud Console에서 얻기

### 3-2. Google Cloud Console에서 설정
1. https://console.cloud.google.com 방문
2. 프로젝트 생성 (없으면)
3. "OAuth 동의 화면" 설정:
   - 앱 이름: "상세페이지_기획_생성기"
   - 승인된 도메인: `supabase.co` 추가
4. "사용자 인증 정보" > "OAuth 2.0 클라이언트 ID" 생성:
   - 유형: "웹 애플리케이션"
   - 승인된 리다이렉트URI:
     ```
     https://xxxxx.supabase.co/auth/v1/callback
     ```
5. Client ID와 Secret을 Supabase에 입력

## 4. Supabase 데이터베이스에 사용자 테이블 생성

Supabase 대시보드 > "SQL Editor"에서 다음을 실행:

```sql
-- users 테이블 (선택사항: 추가 사용자 정보 저장)
CREATE TABLE IF NOT EXISTS users (
  id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email VARCHAR(255) NOT NULL UNIQUE,
  name VARCHAR(100),
  avatar_url TEXT,
  auth_provider VARCHAR(50) DEFAULT 'google',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 데이터 접근 권한 설정 (RLS)
ALTER TABLE users ENABLE ROW LEVEL SECURITY;

CREATE POLICY "사용자는 자신의 정보만 볼 수 있음" ON users
  FOR SELECT USING (auth.uid() = id);

CREATE POLICY "사용자는 자신의 정보만 수정할 수 있음" ON users
  FOR UPDATE USING (auth.uid() = id);

CREATE POLICY "인증된 사용자는 새 데이터를 삽입할 수 있음" ON users
  FOR INSERT WITH CHECK (auth.uid() = id);
```

## 6. 문제 해결

### Q: "NEXT_PUBLIC_SUPABASE_URL is not defined" 에러
→ `.env.local` 파일에 환경 변수를 추가했는지 확인하세요.
→ 파일을 수정한 후 개발 서버를 재시작하세요 (Ctrl+C 후 `npm run dev`)

### Q: Google 로그인 버튼을 눌러도 아무 일이 없음
→ Google Cloud Console에서 OAuth 설정을 확인하세요.
→ 브라우저 개발자 도구 (F12) > Console에서 에러를 확인하세요.

### Q: 로그인 후 대시보드가 안 뜸
→ 브라우저 개발자 도구 > Network 탭에서 요청 상태를 확인하세요.
→ Supabase가 정상 작동하는지 확인하세요.
