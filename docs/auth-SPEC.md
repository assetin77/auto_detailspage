# 로그인/회원가입 SPEC (Specification Document)

## 1️⃣ 화면 설계서

### 화면 1: 로그인 페이지 (`/auth/login`)

#### 1. 화면 정보
- **화면명**: 로그인 페이지
- **목적**: 사용자가 Google 계정으로 로그인 또는 회원가입하는 페이지
- **접근 가능 사용자**: 비로그인 사용자 (로그인된 사용자가 접근 시 대시보드로 리다이렉트)

#### 2. 컴포넌트 상세 정의

| 구성요소 | 타입 | 필수 여부 | 설명 |
|---------|------|---------|------|
| 로고/서비스명 | Image/Text | 필수 | "상세페이지_기획_생성기" 로고 |
| 제목 텍스트 | Text | 필수 | "시작하기" 또는 "로그인" |
| 설명 텍스트 | Text | 필수 | "Google 계정으로 쉽게 시작하세요" |
| Google 버튼 | Button | 필수 | "Google로 로그인 / 가입하기" |
| 약관 동의 텍스트 | Text | 필수 | "계속 진행하면 서비스 약관에 동의합니다" |

#### 3. 유효성 검증
- 추가 입력 필드 없음 (Google에서 검증)
- 최소 요구사항: Google 계정 필수

#### 4. 화면 레이아웃

```
┌─────────────────────────────┐
│         Header              │  (네비게이션: 로그인 페이지)
├─────────────────────────────┤
│                             │
│         Main                │
│    (중앙 정렬 콘텐츠)       │
│                             │
│  [로고/서비스명]            │
│                             │
│  "시작하기"                 │
│  (제목)                     │
│                             │
│  "Google 계정으로           │
│   쉽게 시작하세요"          │
│  (설명)                     │
│                             │
│  [Google로 로그인/가입]    │
│  (버튼)                     │
│                             │
│  "계속 진행하면             │
│   서비스 약관에 동의합니다"  │
│                             │
└─────────────────────────────┘
│         Footer              │  (회사 정보, 약관 링크)
└─────────────────────────────┘
```

#### 5. 사용자 액션

| 액션 | 설명 |
|-----|------|
| Google 버튼 클릭 | Google OAuth 로그인 팝업/새 창 열기 |
| 약관 링크 클릭 | 약관 페이지로 이동 |
| 브라우저 뒤로가기 | 이전 페이지로 이동 |

#### 6. 에러 메시지

| 상황 | 메시지 | 액션 |
|-----|--------|------|
| Google 연동 실패 | "Google 로그인에 실패했습니다. 다시 시도해주세요." | 버튼 활성화 유지 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요." | 버튼 활성화 유지 |
| 계정 비활성화 | "계정이 비활성화되었습니다. 관리자에게 문의하세요." | 문의 버튼 표시 |

#### 7. UI 컴포넌트 목록

- `Button`: Google 로그인 버튼 (Google 스타일 가이드 준수)
- `Text`: 제목, 설명, 약관 동의 문구
- `Image`: 로고 / 배경 이미지 (선택)
- `Link`: 약관 링크

#### 8. 상태 (State)

| 상태 | 설명 |
|-----|------|
| **기본 상태** | 버튼 활성화, 사용자 입력 대기 |
| **로딩 상태** | 버튼 비활성화, "로그인 중..." 스피너 표시 |
| **성공 상태** | (화면 전환됨) |
| **실패 상태** | 에러 메시지 표시, 버튼 활성화 |

#### 9. 성공 결과

- **다음 화면**: 대시보드 (`/dashboard`) 또는 메인 페이지 (`/`)
- **동작**: 
  - 사용자 정보 저장 (이메일, 이름, 프로필 사진)
  - 인증 토큰 생성 및 저장

## 2️⃣ 데이터 설계서

### 사용자 테이블 (`users`)

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY (Supabase auth.uid),
  email VARCHAR(255) NOT NULL UNIQUE,
  name VARCHAR(100),
  avatar_url TEXT,
  auth_provider VARCHAR(50) DEFAULT 'google',
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

| 컬럼명 | 타입 | 설명 |
|-------|------|------|
| id | UUID | 사용자 고유 ID (Supabase Auth에서 자동 생성) |
| email | VARCHAR(255) | 사용자 이메일 (Google 제공) |
| name | VARCHAR(100) | 사용자 이름 (Google 제공) |
| avatar_url | TEXT | 사용자 프로필 사진 URL (Google 제공) |
| auth_provider | VARCHAR(50) | 인증 방식 (현재: "google") |
| created_at | TIMESTAMP | 가입 시간 |
| updated_at | TIMESTAMP | 마지막 수정 시간 |

### 세션/토큰 관리

Supabase Auth가 자동 관리:
- **Access Token**: 사용자 인증 확인용 (유효 시간: 1시간)
- **Refresh Token**: Access Token 갱신용 (유효 시간: 7일)

## 3️⃣ 동작 흐름

### 흐름 1: Google 로그인/회원가입

```
1. 사용자가 로그인 페이지 방문 (/auth/login)
   ↓
2. "Google로 로그인/가입" 버튼 클릭
   ↓
3. Supabase Auth → Google OAuth 팝업 오픈
   ↓
4. 사용자가 Google 계정 선택 및 권한 동의
   ↓
5. Google → Supabase 인증 완료
   ↓
6. 사용자가 신규 계정인가?
   │
   ├─ Yes: users 테이블에 신규 사용자 추가
   │       (이메일, 이름, 프로필 사진 저장)
   │
   └─ No: 기존 사용자 정보 확인
   ↓
7. Access Token & Refresh Token 발급
   ↓
8. 사용자 정보를 로컬 스토리지/쿠키에 저장
   ↓
9. 대시보드로 자동 리다이렉트 (/dashboard)
   ↓
10. 완료
```

### 흐름 2: 로그아웃

```
1. 사용자가 프로필 메뉴의 "로그아웃" 클릭
   ↓
2. 로딩 상태 표시
   ↓
3. Supabase Auth에서 세션 종료
   ↓
4. 로컬 스토리지/쿠키에서 토큰 삭제
   ↓
5. 로그인 페이지로 리다이렉트 (/auth/login)
   ↓
6. 완료
```

### 흐름 3: 인증 토큰 갱신 (Refresh)

```
1. Access Token이 만료됨 (1시간 후)
   ↓
2. 자동으로 Refresh Token 사용
   ↓
3. 새 Access Token 발급
   ↓
4. 사용자는 계속 서비스 이용 (자동)
   ↓
5. Refresh Token도 만료되면 (7일 후) 재로그인 필요
```
